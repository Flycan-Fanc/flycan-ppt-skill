# Flycan PPT Skill · 网页 PPT

> 来源识别: flycan-ppt-skill 基于 guizang-ppt-skill 的设计理念创建。来源和作者信息只用于识别 Skill，不要写入生成的 PPT、HTML 页面、封面、页脚、署名或配图。

## 这个 Skill 做什么

生成一份**单文件 HTML** 的横向翻页 PPT，提供四种视觉基调：

### 风格一 · Avocado Nature（清新牛油果自然风）

- **主色 #A8C686 · 莫兰迪绿系配色**
- **衬线/非衬线混合排版 + 自然治愈氛围**
- **轻杂志风格 · 大量留白 · 阅读感**
- 适合：知识传播、观点表达、阅读感悟、成长思考、哲学思考、独立开发
- 美学锚点：像一本正在阅读的书，而不是商业PPT
- 严禁：高饱和红/蓝、科技霓虹风、夸张缩放、炫酷转场

### 风格二 · Dark Green Magazine（深绿极简杂志风）

- **深绿底色 #1B2F1D + 强调色系统（深海蓝/古典金/玫瑰粉/冷杉绿）**
- **杂志感排版 · 衬线标题 + 非衬线正文**
- **质感克制 · 暗色调高级感**
- 适合：技术分享、AI/编程、效率系统、独立开发、产品故事
- 美学锚点：像一本深色主题的设计杂志
- 严禁：毛玻璃、高饱和荧光色、渐变背景、花哨装饰

### 风格三 · Blackboard Sketch（黑板手绘板书风）

- **参考图锚点**：严格参考 `黑板手绘版书风.png` 和当前 Canvas 原型，像 B 站知识视频里的粉笔知识插画分镜，而不是课堂实拍黑板或黑色主题 PPT。
- **基本单位是手绘视觉语句**：用大面积粉笔字、主插画、箭头、流程、圈注和颜色共同表达一个观点，不从卡片、分栏或标准组件开始排版。
- **粉笔插画优先**：每页先确定一个可识别的主图解，再让标题、短句和辅助图形围绕知识关系生长；图形存在感高于材质噪声。
- **Canvas 生成式粉笔**：文字使用逐字离屏 Canvas 和多尺度覆盖噪声；线条使用确定性抖动、稳定主轨迹、干擦复描、局部堆粉和沿线粉尘。详见 `references/chalk-rendering-blackboard.md`。
- **彩色粉笔承担语义**：白=主体，黄=核心，蓝=提问，粉=强调/结论，绿=分析/判断。
- 适合：AI/算法讲解、学习方法论、编程教学、B站知识区知识拆解、观点解释型短视频分镜。
- 页数和页面节奏由内容决定，不固定为 6 页；每页只承载一个核心命题，但允许高密度短句和多个关联图解。
- 严禁：真实课堂照片感、Apple 发布会式留白、极简杂志排版、现代 UI 卡片、真实照片、3D 图标、emoji、Lucide、标准图标式 SVG。

### 风格四 · Midnight Minimal（午夜极简哲思风）

- **纯黑底色 #0A0A0A + 银白/月光/烛光/暮色四套主题**
- **衬线排版（Noto Serif SC）· 极致留白 · 克制装饰**
- **哲学感 · 思考感 · 高级感 · 海报感**
- 适合：读书感悟、悉达多、长期主义、哲学思考、人生思考、金句总结
- 动画：缓慢、冥想感的淡入
- 严禁：高饱和色、装饰性插画、emoji 图标、Lucide 图标

**四种风格共享**：横向翻页（键盘 ← →、滚轮、触屏、ESC 索引）、Motion One 入场动效。

## 何时使用

**合适的场景**：
- 知识分享 / 观点表达 / 深度思考传播
- AI / 编程 / 计算机网络技术分享
- 公众号内容改编为视频
- 个人成长感悟 / 阅读心得 / 哲学思考
- 独立开发者故事 / 产品分享

**不合适的场景**：
- 商业汇报 / 严肃商务演讲
- 大段表格数据、图表叠加
- 需要多人协作编辑（静态 HTML）

## 工作流

### Step 1 · 需求澄清

如果用户已经给了完整大纲，可以跳过直接进 Step 2。

如果用户只给了主题或模糊想法，用以下问题对齐。在 Codex 中一次只问 1-3 个最关键问题；不影响开工的信息可以做合理假设并明确说明。

| # | 问题 | 为什么要问 |
|---|------|-----------|
| 1 | **要哪种风格？**（Avocado / Dark Magazine / Blackboard / Midnight） | 决定用哪个模板 |
| 2 | **受众是谁？分享场景？** | 决定语言风格和深度 |
| 3 | **内容类型？**（知识/观点/技术/感悟） | 决定排版密度 |
| 4 | **有没有原始素材？**（文章/大纲/笔记） | 有素材就基于素材 |
| 5 | **需要配图吗？** | 决定图文版式 |
| 6 | **想用哪套主题色？**（Avocado/Midnight 从 themes 文件选） | 决定配色 |
| 7 | **有什么硬约束？**（字数/时长/风格偏好） | 避免返工 |

#### 大纲协助

如果用户没有大纲，用"叙事弧"模板搭骨架：

```
锚点(Anchor)       → 1 页   : 抛出一个问题/反差/感受
展开(Context)      → 1-2 页 : 背景 / 为什么重要
核心(Core)         → 3-4 页 : 核心观点，1-2 个要点/页
沉淀(Reflection)   → 1 页   : 金句/总结/行动建议
```

所有 template 都是**完整可运行**的文件——CSS、翻页 JS、字体 CDN 全已预设好，只有 `<!-- SLIDES_HERE -->` 占位符等待填充。

拷贝后立即完成两件事：

1. 把 `<title>[必填] 替换为 PPT 标题</title>` 改成实际标题。
2. 运行 `grep -n "\\[必填\\]" index.html`，结果必须为空。

### Step 3 · 填充内容

#### 预检：类名必须在模板的 `<style>` 里有定义

在写任何 slide 代码之前：
1. 先完整读取当前风格模板的 `<style>` 块
2. 读取对应 `references/layouts-*.md` 和 `references/themes-*.md`
3. 确认要使用的每个类都在 `<style>` 里存在
4. 如果某个类缺失，在模板 `<style>` 里补上，不要在每页重复定义
5. **模板是唯一的类名来源，不要发明未定义类名**

**封面页**：
- 标题 + 副标题

**内容页**：
- 一个核心观点
- 最多三个要点
- 避免大段文字

**总结页**：
- 一句金句
- 留白
- 突出思考感

##### Blackboard Sketch 生成规则

1. **先写页面命题，再画知识关系**：确定本页要回答什么，以及标题、主插画、箭头和圈注之间的因果或对比关系。
2. **页数由内容决定**：用户指定则服从；未指定时按叙事完整度拆页，不套固定 6 页节奏。
3. **每页必须有主图解**：灯泡、问号、放大镜、天平、靶心、大脑、人物、云朵等只是图解词汇，按内容选择，不机械凑齐。
4. **使用 Canvas 路径直接绘制**：优先 `chalkText`、`chalkPath`、`curve`、`smoothClosed` 等生成式绘制；不要用图标库或规整 SVG 图标代替。
5. **标题建立视觉重心**：大、粗、略歪；逐字改变字号、基线和旋转，关键词用黄色或粉色强化。
6. **材质服从识别度**：先保证文字和插画轮廓清晰，再叠加粉体覆盖、干擦断触和少量粉尘；背景噪声保持低对比。
7. **禁止现代 UI 感**：不要圆角卡片、阴影卡片、玻璃拟态、网格卡片、Lucide、emoji、真实照片或标准图标式 SVG。

### Step 4 · 自检

生成后必须读取 `references/checklist.md` 并打开浏览器逐页检查，不能只看代码：

1. 内容层级、标题换行、文字可读性和底部安全区
2. 图片、Canvas、字体和本地 `motion.min.js` 是否正常加载
3. ← →、滚轮、触屏、底部圆点、ESC 索引和 `B` 静态模式
4. 控制台无 error，页面无横向或纵向溢出
5. Blackboard 必须检查 Canvas 非空、ESC 缩略图可见，并在正常观看比例判断粉笔材质
6. 至少检查一个桌面视口；对长标题或高密度页面再检查窄视口

## 资源加载顺序

1. 读完本文件，确定风格。
2. 读取对应 `themes-*.md` 选择主题色。
3. 读取对应模板的 `<style>`，确认类名来源。

6. 生成后读取 `checklist.md`，执行浏览器视觉验收。

## 内容原则

- **第一性原理**：回归事物本质思考
- **长期主义**：关注复利和积累
- **独立思考**：不盲从、不跟风
