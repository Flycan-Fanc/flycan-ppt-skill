# 质量检查清单（Checklist）

生成 PPT 前，先通读一遍；生成后，逐项自检。

---

## 🔴 P0 · 一定不能犯的错

### 1. 预检：类名必须存在

**现象**：直接粘贴 layouts 骨架，结果样式全部丢失——大标题字体不对、布局错乱。

**做法**：
- 生成 PPT 前，先 Read 当前模板的 `<style>` 块
- 确认 layouts 里用到的类都已定义
- 如果某个类缺失，在模板 `<style>` 里补上

### 2. 不要用 emoji 作图标

用 Lucide 图标库。

### 3. 页面主题节奏

- 不要连续 3 页以上相同主题（light 或 dark）
- Avocado 风：hero 和 non-hero 交替
- Blackboard 风：所有页用 dark 主题，但可以用不同布局制造节奏

### 4. 标题字号不能超过屏宽

- `h-hero` 最大 10vw，标题长度 ≤ 5 字
- 长标题用 `<br>` 手工断行

### 5. 字体分工

| 角色 | Avocado | Blackboard |
|------|---------|------------|
| 标题 | Noto Serif SC（衬线） | Noto Serif SC / ZCOOL XiaoWei |
| 正文 | Noto Sans SC（非衬线） | Noto Sans SC |
| 元数据 | IBM Plex Mono（等宽） | JetBrains Mono |

### 6. 图片不要用原图奇葩比例

用标准比例 16:10 / 4:3 / 3:2 / 16:9，`object-fit:cover` 做裁切。

### 7. chrome 和 kicker 不要写同一句话

- `chrome` = 杂志页眉 / 导航标签（可跨页重复）
- `kicker` = 本页独一份的引导句（每页不同）

### 8. 动效标注

- 所有页至少给标题/正文/主要元素加 `data-anim`
- 金句页用 `data-animate="quote"`
- 对比页用 `data-animate="directional"`
- 流程页用 `data-animate="pipeline"`

---

## 🟡 P1 · 排版节奏

### 9. Hero 页和非 hero 页要交替（Avocado 风）

推荐节奏：封面(hero) → 内容页 × 3-4 → 幕封(hero) → 内容页 × 3-4 → 金句页 → 总结

### 10. 留白要充足

Avocado 风以留白为特色，不要塞满内容。内容页最多 3 个要点。

### 11. 术语统一

全篇统一用词，不要中英混译。

### 12. 每页一个核心观点

避免一页塞多个观点。如果信息太多，拆成多页。

---

## 🟢 P2 · 视觉打磨

### 13. Blackboard 风：黑板质感的保持

- 不要在白底上使用黑板元素
- 粉笔字颜色：白为主、黄高亮、蓝辅助、红强调
- 分割线用极淡灰白

### 14. Avocado 风：配图风格

- 纪实感、自然光风格
- 避免：高饱和、滤镜过度、合成感强
- 主色不出 #A8C686 和辅助色范围

### 15. 动画克制

- Avocado：轻微淡入 + 轻微位移，不要夸张缩放
- Blackboard：逐步出现，像老师写板书

---

## 🔵 P3 · 操作细节

### 16. 图片路径用相对路径

`images/xxx.png`，不要用绝对路径。

### 17. 页码写死

JS 动态算总页数，但 `.chrome` 里的 `XX / N` 是写死的。

### 18. 翻页导航要保留

模板默认支持：← → / 滚轮 / 触屏滑动 / 底部圆点。不要删 JS 里的导航逻辑。

### 19. `<title>` 要改为实际标题

grep "\[必填\]" 应无结果。

## 最终自检清单

```
预检
  □ 已读过 template 的 <style>,确认所需类都存在
  □ 已决定每页用哪个 Layout
  □ 已画出页面节奏表
  □ <title> 已改为实际 deck 标题

内容
  □ 每页一个核心观点
  □ 最多三个要点
  □ 避免大段文字
  □ 没有使用 emoji 作图标

排版(仅 Avocado)
  □ 标题 vs 正文间距 ≥ 4vh
  □ hero 页和 non-hero 页交替
  □ 留白充足

排版(仅 Blackboard)
  □ 所有页使用 dark 主题
  □ 粉笔配色正确（白/黄/蓝/红）
  □ 黑板质感不破坏

动效
  □ 翻页时内容逐个淡入
  □ 金句页有逐行揭示效果
  □ 流程页支持手动逐步推进
  □ data-anim 数量 ≥ 页数 × 3

交互
  □ ← → 翻页正常
  □ 底部圆点数量与总页数匹配
  □ chrome 页码和实际页号一致
```
