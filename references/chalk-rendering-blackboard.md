# Blackboard Canvas 生成式粉笔渲染

本规范记录 `output/chalk-canvas-prototype/index.html` 的当前实现，作为 Blackboard 后续粉笔材质改进的算法基线。

> 当前方案已确定生成路径和视觉方向，但正常观看比例下的粉笔沉积、断触和毛糙边缘仍需继续增强。不要把“代码可运行”写成“粉笔视觉已验收”。

## 使用原则

- 使用固定逻辑画布 `1536 × 864`，按设备像素比缩放，`devicePixelRatio` 上限为 `2`。
- 所有随机扰动必须由 seed 驱动；同一页面每次重绘保持一致，避免 resize 后跳动。
- 先保证主字形和插画轮廓清晰，再添加粉体缺口、干擦复描和粉尘。
- 标题允许更强的粉体变化；正文提高连续覆盖率，避免为了材质牺牲可读性。
- 图形存在感高于材质噪声；背景只保留低对比擦痕和少量颗粒。

## 核心 API

### `rng(seed)` 与 `noise2(x, y, seed)`

- `rng` 提供可复现的伪随机序列，用于逐字旋转、基线漂移、路径抖动和粉尘位置。
- `noise2` 提供离散二维噪声，用于文字像素覆盖率。
- 每个页面、文字和图形使用稳定且互不相同的 seed。

### `chalkText(text, x, y, size, color, seed, options)`

逐字绘制，不直接把整句文字一次性画到主 Canvas。

1. 为每个字符创建离屏 Canvas。
2. 使用手写字体回退栈绘制稳定字形：
   `"HanziPen SC","Hannotate SC","Kaiti SC","STKaiti",cursive`
3. 读取字符 alpha，叠加三种尺度的覆盖变化：
   - 细纤维：约 `3 × 1.4px`
   - 中纤维：约 `8 × 3px`
   - 粉体沉积：约 `13 × 11px`
4. 标题使用较低且变化更明显的覆盖率；正文使用更高、更连续的覆盖率。
5. 只在字形覆盖区域附近添加少量椭圆粉体颗粒。
6. 将字符逐个合成到主 Canvas，并对字号、基线、旋转和字距做小幅 seed 扰动。

标题与正文不能共用同一材质强度：

| 层级 | 目标 |
|---|---|
| 大标题/关键词 | 保留稳定字形，同时看见中尺度浓淡、局部堆粉和轻微缺粉 |
| 正文/标签 | 连续覆盖优先，降低噪声幅度和粉尘数量 |
| 页码/署名 | 小字号高覆盖，避免颗粒破坏识别 |

### `chalkPath(points, color, width, seed, closed, passes)`

用于箭头、星星、图标轮廓、下划线和折线路径。

1. 第一遍绘制高覆盖主轨迹，保留形状识别度。
2. 第二遍使用更细线宽、较低透明度和不规则 dash，形成干擦复描。
3. 随机选择少量路径段添加短段粉体堆积。
4. 沿采样点附近添加低透明度椭圆粉尘。
5. 保持 `round` line cap / join，不使用锐利机械端点。

不要通过改变原始轮廓、空间关系或线宽层级来制造材质。

### `curve(points, ...)`

- 使用三次贝塞尔控制点采样为短折线，再交给 `chalkPath`。
- 用于手绘横线、弯箭头和非规则连接线。
- 不直接依赖浏览器平滑描边作为最终材质。

### `smoothClosed(points, ...)`

- 用二次曲线中点法连接闭合点集。
- 两遍复描分别承担主轮廓和干擦断触。
- 用于云朵、脑袋、对话区域等不规则闭合插画。
- 允许轻微不对称和不闭合感，禁止变成规整圆角容器。

## 背景

- 黑板底色使用接近 `#101411` 的深绿黑。
- 使用少量低透明径向擦痕和稀疏颗粒。
- 背景纹理不能与文字粉尘竞争，也不能模拟真实课堂摄影。
- 细边界和页码可保留，但不是每页构图的起点或强制视觉中心。

## 配色语义

| 颜色 | 当前原型 | 语义 |
|---|---|---|
| 黑板 | `#101411` | 低调画布 |
| 白 | `#FFF9E9` | 主体文字与轮廓 |
| 黄 | `#FFD52F` | 核心词、关键判断 |
| 蓝 | `#4DC8FF` | 问题、提示、探索 |
| 粉 | `#FF6294` | 强调、转折、结论 |
| 绿 | `#89E052` | 分析、判断、正向关系 |

## 构图方式

- 每页先确定一个核心命题和一个主插画。
- 用标题尺度、颜色和空间邻近建立阅读顺序，不依赖标准网格组件。
- 让箭头连接观点，让云朵承载补充辨析，让图标序列承担推导。
- 同一页可以同时出现主标题、主插画、流程、圈注和结论，但必须共享一条知识关系。
- 页数由内容决定，不固定为 6 页。

## 验证

- 检查 JavaScript 静态语法。
- 在桌面和窄视口截图，确认画布完整缩放且无溢出。
- 检查每个 Canvas 都有非透明像素。
- 检查控制台 errors / warnings。
- 视觉验收时使用正常观看比例，不以放大后的局部颗粒代替整体粉笔感判断。
